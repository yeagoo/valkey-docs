Valkey 是 Redis 7.2.4 的 BSD-3 开源分叉，由 Linux Foundation 托管，背后有 AWS、Google、Oracle、Ericsson、Snap 等共同支持。它在协议与数据格式上兼容 Redis 7.2，因此各大云厂商几乎都在 2024 年底到 2025 年间推出了托管 Valkey 服务。本页给你一个快速全景，再带你进入每个厂商的细节页与[选型与成本](/docs/cloud/selection)。

## 为什么选托管而不是自建

自建 Valkey 看似省钱，但你要自己负责高可用、故障切换、版本升级、备份、监控与安全补丁。托管服务把这些都交给云厂商，并且在超大规模运营下，**托管 Valkey 在缓存场景普遍比托管 Redis 更便宜**（AWS 节点型低约 20%，Serverless 低约 33%）。除非你有非常特殊的部署需求，否则生产环境建议直接用托管服务。

<Callout type="info">
许可证层面，Valkey 采用 BSD-3，法律风险最低；而 Redis 自 7.4 起转为 RSALv2 / SSPLv1（外加可选 AGPLv3），属于带 copyleft 倾向的源可用许可。这也是很多团队迁移到 Valkey 的核心动机之一。
</Callout>

## Valkey 是什么，为何各云都支持它

Valkey 诞生于 2024 年 Redis 许可证变更之后，是 Redis 7.2.4 的直接分叉，托管在 Linux Foundation 旗下，治理中立。它的几个关键事实：

- **协议与数据兼容 Redis 7.2**：现有客户端、SDK、运维工具几乎无需改动即可对接。
- **背后阵容强**：AWS、Google、Oracle、Ericsson、Snap 等共同投入，确保长期演进。
- **版本节奏稳**：最新版本为 9.1（2026 年 5 月发布），功能持续向前。

正因为兼容性好、治理中立、阵容强，几乎所有主流云厂商都在一年内推出了托管 Valkey，让"换引擎"几乎是无痛的事。

## 性能与成本的双重红利

迁移到 Valkey 通常不是"为了省钱牺牲性能"，而是两者兼得。第三方基准（Valkey 8.1 vs Redis OSS）显示吞吐高约 8%、P99 延迟低约 22%、内存占用低约 20%；同时各云的托管报价也更低。详细数字与注意事项见 [选型与成本](/docs/cloud/selection)。

<Callout type="info">
现实案例：Snap 将约 70% 集群迁移到 AWS ElastiCache for Valkey 后，缓存支出下降约 60%。这类规模化收益是很多团队启动迁移的直接动因。
</Callout>

## 三种计费/部署模型

理解这三类模型，能帮你快速判断该看哪个服务：

<Cards>
  <Card icon={<Cloud />} title="Serverless（无服务器）">
    无需选规格，自动按量伸缩，一分钟内就绪。流量波动大、不想做容量规划时首选。代表：AWS ElastiCache Serverless。
  </Card>
  <Card icon={<Server />} title="Node-based（节点型）">
    你选择实例规格（如 cache.r7g），价格更可预测，适合稳定负载与成本优化。代表：ElastiCache 节点型、GCP Memorystore。
  </Card>
  <Card icon={<Database />} title="Durable Primary（持久化主库）">
    多可用区持久化的内存数据库，可作为系统的"记录源"（system of record）。代表：AWS MemoryDB for Valkey，99.99% 可用性。
  </Card>
</Cards>

简单的判断口诀：**纯缓存用 ElastiCache，把 Valkey 当持久主库用 MemoryDB。** 详见 [AWS 详情页](/docs/cloud/aws)。
